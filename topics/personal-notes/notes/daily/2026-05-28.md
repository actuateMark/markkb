## Notes / Learnings

- **`label.deployment_phase` is on `K8sContainerSample` only, not on `Log`.** Burned ~10 min discovering Log queries scoped by `label.deployment_phase = 'staging'` silently return 0 rows. Workaround for stage-fleet log scoping: enumerate container names from `K8sContainerSample` first, then filter Log queries by `container_name IN (...)`.
- **NR `Stream lost: [Errno 1414092869] Immediate exit requested` ≠ `broken stream`.** The avigilon soak site logged 5 stream-disconnect ERRORs in 14h that my initial `message LIKE '%broken stream%'` grep didn't catch (different message wording). Worth adding `'Stream lost'` to the standard reconnect-pattern alternation. The Errno suggests an HTTP/connection error from the Avigilon API, not a PyAV-level issue.
- **Stage CrashLoop top offenders are pre-existing settings-load orphans.** `connector-705-fs-630`, `connector-44300-fs-{1466,1468,1464,1470}`, `adpro-puller` — all crashing with `Failed to load settings file, closing without retry`, none with blacklist activity. These are feature-scope orphan pods, unrelated to the blacklist decay lib bump (or the PyAV soak). Worth a separate cleanup pass if they're long-stale.
- **`gh pr edit` silently fails on vms-connector (classic-Projects deprecation).** Editing PR #1713's title + body via `gh pr edit --title ... --body-file ...` returned a non-fatal GraphQL deprecation notice about "Projects (classic)" — but the mutation itself didn't land (title and `updatedAt` unmoved). Workaround that works: `python3 -c "import json; print(json.dumps({'title':..., 'body':...}))" | gh api -X PATCH repos/aegissystems/vms-connector/pulls/<n> --input -`. The REST PATCH bypasses the GraphQL field that's choking. Worth memoing for any future PR-body edit on this repo.
