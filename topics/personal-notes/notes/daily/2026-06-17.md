### onboarder: merge-to-master auto-deploys the prod Lambda
- `autopatrol_onboarder/.github/workflows/deploy.yml` triggers `on: push: branches: [master]` → **every merge to `master` is a prod Lambda deploy.** Default branch is `master` (not `main`); PRs target `master`; manual `deploy_prod*.sh` scripts also exist. Resolved the previously-"unknown — ask" row in [[branch-conventions]].

### squash-body CI-skip guard (#1755) is a conservative screen, not a literal directive detector
- The guard's regex (`no[-_ ]?ci|ci[-_ ]?skip|…`, inherited from the global rule) flagged onboarder #15's benign prose **"no CI changes needed"** — a **false positive**: GitHub only suppresses on the *bracketed/asterisked* forms (`[no ci]`, `[skip ci]`, `[ci skip]`, `***NO_CI***`, `[skip actions]`, `[actions skip]`), not bare substrings. The screen worked as intended (flag → human judgment → clean the body before merge). **Possible follow-up:** tighten the guard to the bracketed forms to cut false-positive noise (alert-fatigue risk). For now, treat hits as review prompts, not hard blocks.
