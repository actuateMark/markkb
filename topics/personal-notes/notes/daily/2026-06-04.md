### Verifier reevaluation — how the 80-vs-10 discrepancy resolved (tooling)

- **Tooling path that settled it:** prod AWS SSO (`388576304176`, AdministratorAccess) → `aws s3 cp s3://actuate-settings/connector-34952/settings.json`. US settings bucket = `actuate-settings` (`region_config.py:71`); key layout `<connector_id>/settings.json` (deployed) + `<connector_id>/preview/settings.json` (`settings_generator.py:764`). EU bucket = `actuate-eu-settings-prod-eu-west-1`.
- **S3 only holds the post-override output** — the raw `minimum_confidence` the user sees on the change form lives only in the admin DB. Both "80" (our generated snapshot, Medium fixture) and "10" (raw form field) are real but neither is the live gate; deployed = **40 / 1-of-1** for 396242 today.
- **Key gotcha worth remembering:** editing the raw `minimum_confidence`/`thresh`/`denominator` on a camera does **nothing** while a Sensitivity is selected — the fixture wins. To deploy a custom gate you must clear Sensitivity (raw passes through) or pick the Sensitivity tier whose fixture you want.
- **Camera map for connector-34952** (automation test site, "DO NOT TOUCH"): 395216=intruder A, 402535=intruder B, 402762=intruder C, **396242=pistol A (High, 40/1-of-1)**, 396322=bike, 396243=vehicle A, 470940=vehicle B.
