## Notes / Learnings

### #1731 prod merge — CI-skip token suppressed the build/deploy (incident + recovery)

- **Recurrence of the squash-body CI-skip suppression (2nd time; first was PR #1688 2026-05-11).** #1731 squash-merged as `d0066b43`, but the default squash body (line ~375) carried a literal CI-skip directive inside a backtick-quoted hygiene phrase — GitHub Actions parses these *anywhere in the message*, so it silently suppressed all push-triggered rearch workflows (build/deploy, sonar, reset-stage). The merge landed but **no image built / no deploy happened** (rearch.yml's last run was 06-09, not d0066b43).
- **Recovery:** jacob pushed empty commit `11ff4d6c` + `gh workflow run rearch.yml --ref rearchitecture` (workflow_dispatch bypasses commit-message suppression; no force-push). ARM64 + x86 + deploy all green. Full write-up: [[2026-06-16_squash-merge-ci-skip-suppression-recurrence]].
- **Prevention shipped:** added a BLOCKING "scan the *assembled* squash body for CI-skip directives" step to `vms-connector/.claude/agents/pr-prep.md` (4.5) and `.claude/skills/pre-merge-workflow.md` (Step 4) — greps `git log --format='%B' rearchitecture..HEAD` for all token variants and forces a cleaned explicit `--subject`/`--body`. Key lesson: scan the *assembled* body (token leaks via concatenated prose), verify at *merge* time not just prep, and recovery is always workflow_dispatch (a CI guard can't catch this — it gets suppressed too). *(edits in working tree on `stage`, not yet committed)*
