## Notes / Learnings

### #1703 — PyAV 17.0.1 skip_frame finding (scope shrink)
The issue body claims the 7 `codec_context.skip_frame = "DEFAULT"/"NONKEY"` sites must move to `SkipType` enum members on PyAV 17. **Not true for PyAV 17.0.1** — `from av.codec.context import SkipType` raises ImportError (the enum is gone from the documented path) and `skip_frame` is still a `str` subclass that accepts the same strings as av13. The real migration was just `av.AVError → av.FFmpegError` (av17 dropped AVError; av13.1 has both). The compat shim's string fallback handles av17 transparently. Full detail: [[2026-05-26_pyav17-local-validation]].

### §29 — reboot_connector short-circuits on `settings.STAGE`, NOT customer phase
`ConnectorController.reboot_connector` (inframap/connector/connector_controller.py:422) gates the real ECS/EKS reboot on `if settings.STAGE != "prod": ...fake reboot; return`. That's the **admin server's environment**, not the customer's `deployment_phase`. Consequence: deploying a custom branch to a STAGE-phase test customer **via prod admin** issues a *real* reboot (scoped to that customer). The deploy_branch docstring's "short-circuits on non-prod STAGE" refers to running the staging/local admin server (where §29 was locally verified) — which is why the would_deploy payload had to be built inline there. So a cust-705 dry-run via prod admin genuinely exercises the av17 runtime on a real connector, with blast radius contained to an Actuate-owned test site.

### deploy_branch endpoint contract (for the dry-run)
`POST /api/customers/{id}/deploy_branch/` — superuser only. Body: `image_tag` (required), `reason` (required), `expires_at` (optional ISO-8601, default +7d). Flips customer to CUSTOM phase, snapshots `pre_custom_deployment_phase`, creates a `ConnectorVersion`, writes a `BranchDeploymentEvent` audit row, calls `reboot_connector`. Reverts via `POST .../revert_branch/` (body: `reason`). The connector image for #1714 is in ECR as `featurepyav-17-bump-clean`.
