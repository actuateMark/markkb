### Session-wrap follow-ons (CI fix + dev-verify + prod PR open)

- **Test step CI failure on PR #90 → root cause + fix.** Pytest's default collection pattern matches `*_test.py`. Two of the new verification scripts (`v5_motion_image_test.py`, `v5_motion_pertag_test.py`) end in `_test.py`, so when tox ran `pytest --ignore=tests/integration` from the repo root, it tried to *import* them at collection time. Both scripts call `sys.exit(2)` at module top-level when `V5_VERIFY_API_KEY` is unset (which it is in CI) → SystemExit aborts collection → tox exits 3 → CI red. Fix: added `--ignore=tests/verification` to the tox pytest invocation, corrected the README claim that the scripts were "excluded by directory" (they weren't — they need the explicit ignore). Commit `0555a13`, local tox green (209 passed), CI rerun passed. **Lesson for future verification-script lifts:** if a script's filename matches `*_test.py` or `test_*.py`, it WILL be collected by default pytest regardless of where it lives in the tree — must be explicitly ignored or renamed.
