### From the §12j build (transferable beyond this workstream)

- **Vulture `--min-confidence 80` is the right default for Django-shaped repos**, not the documented 60%. At 60% Django magic (Meta classes, model fields, migrations, apps.py, enum members) dominates output: `actuate_admin` 3308 findings, `actuate_monitoring_api` 499. At 80% they drop to 41 / 6 — a 98% cut while preserving rank order across all 16 repos. Lesson: when a lint tool's default produces noisy FPs on idiomatic framework code, fix at the source (config) rather than per-repo allowlist patches. This is the same architectural choice as Pattern A (single source class) over Pattern B (per-repo overrides) elsewhere in the same workstream.
- **`subprocess` PATH-inheritance trap on minipc/cron**: tools installed via `uv tool install` to `~/.local/bin/` work under the cron entrypoint (which exports PATH) but fail under ad-hoc SSH (non-login shell, bare distro PATH). Module-level `os.environ['PATH']` prepend in `collect.py` covers both invocation paths transparently. Worth checking any `subprocess.run` that calls a user-local binary.
- **FastAPI behind Caddy `handle_path` strip needs explicit `servers:`** to make Swagger UI "Try it out" route correctly. With `root_path="/app"` and `handle_path /app/api/*`, the OpenAPI default publishes operations at `/<route>` with no prefix, so the browser-side fetcher hits a non-existent external URL. Setting `servers=[{"url":"/app/api"}]` makes the spec self-describe its actual external prefix. Pair this with a custom `/api-docs` route hard-coding `openapi_url="/app/api/openapi.json"` since the auto-mounted docs assume `<root_path>/openapi.json` (which falls outside the strip handler).
- **"Drop None from FACET" as a signal-design pattern**: when a per-repo metric is N/A (e.g. unpinned package, no open PRs, no CI runs), the collector should drop the repo from the FACET dict rather than emit `None`. Keeps signals focused on repos with real data. Applied consistently to `repo_actuate_*_pin`, `repo_open_prs_p50_age_days`, `repo_ci_failure_rate_pct` — same pattern, four signals.
- **Sink JSONL as a pre-aggregated read-cache** for the `/app/repos/` 11ty page: `prebuild.js` reads `~/Documents/worklog/dashboard/sink/observations.jsonl`, builds a latest-per-signal map, and re-keys by repo. This means *any* dashboard-check signal automatically becomes available to *any* static-gen page that wants it — no per-page wiring beyond a 30-line helper. Same architecture would let `/app/today/` or future detail pages consume signals with zero new infrastructure.
