## Notes / Learnings

- **Multi-subagent fan-out pattern is proven.** Two independent pilots today (K8s reading list + frame-storage reading list). In both, 3 parallel subagents × 3 entries each produced structured proposals in ~3-5 min wall-clock. Source-reader role is the cheapest + most reliable; research-prospector needs calibration work on the quality rubric (ambiguous between 3 and 4 for vendor docs) and PDF/redirect fallback logic (now encoded in the agent spec).
- **Concept dedup requires a main-agent pass.** Subagents can't coordinate across batches, so overlapping concept proposals emerge (e.g. `kvs-fragment-model` + `fragment-duration-api-call-amortization`, `edge-compress-before-cloud-sync` + `vms-pre-buffer-pattern`). Dedup is mechanical but has to happen serially post-batch. Synthesizer role will own this in the formal pipeline once built.
- **PDF workaround works cleanly for TOC'd PDFs.** `curl -sSL -o _research-inbox/{slug}.pdf {url}` + `Read(file, pages="1-N")` reads page renders. Verified on Milestone (48-page PDF). Limitation: content comes back as rendered images, not extractable text — visual reading, not grep-able. OK for summary extraction; would break down for search-across-PDFs workflows.
- **Cost structure reframes the frame-storage problem.** "S3 spend is dominated by API calls, not data volume" — if accurate, the per-frame PUT pattern is the primary cost driver, not the byte count. Current pattern is ~22 S3 calls per 10-frame window. In-process video encoding drops that to ~2 calls. In-cluster blob + conditional promotion drops non-eventful windows to 0 calls. **Potential >50× API-call reduction** for typical always-on camera fleets. This reframes the video-reconstruction idea from "save storage" to "collapse N GETs into 1" — a much bigger lever.
- **Milestone's VMS pre-buffer = exactly our §12 design.** Their white paper (2023-09, John Rasmussen) describes a 15-second RAM buffer, event-triggered flush to recording DB, discard-if-no-trigger, with an 80% storage-load reduction at 20% recording rate. This is the external validation that memory-first-plus-conditional-flush is a viable architecture pattern at commercial VMS scale.
- **KVS warm tier validates the fragment-amortization economics but is architecturally incompatible with conditional promotion.** Stream-wide static tier + 30-day retention floor rule it out as a drop-in for §12. The fragment-duration cost table (2s→28%, 5s→48%, 20s→57% savings; break-even at 1.06s) is the quantitative external proof for §11's API-call-dominance thesis though.
- **Connector already has structural advantages for the shift.** Pipeline is stateless w.r.t. frames (only `frame_id` + timestamps flow between steps; pixels live in a central `image_cache`). Clip assembly is already externalized to a `/create-video` lambda. Shifting to clip-based storage is mostly a change in the `/create-video` contract + S3 key shape — connector itself barely changes. The surprise was how clean the current architecture is relative to the disruption we'd feared.
- **R&D-agent output format matters more than prompt content.** Both pilots' subagents returned strict markdown structures with predictable section headers (Source Notes / Concept Proposals / Cross-References / Fetch Log / Open Questions). This made the main-agent merge mechanical. When we build the synthesizer role, insist on the same structural discipline — it's what makes parallel fan-out composable.
- **Today's daily-wrap triggered all three new primitives we built yesterday + today.** Morning-reminder cron + SessionStart hook (yesterday's primitive); kb-recap skill inside daily-wrap Step 1.5.A (today's primitive); `_research-inbox/` + PDF workaround as a source path (today's primitive). All worked. Worth noting how quickly the infra layers up: primitive at wrap-time yesterday → formalized skill today → used in wrap-time tonight.
