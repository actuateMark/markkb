# Janus: general-purpose WebRTC server

[[webrtc-deep-dive|WebRTC]] server developed by Meetecho. Designed to be general-purpose: Janus itself does not provide any application functionality — it only implements the means to set up [[webrtc-deep-dive|WebRTC]] media sessions with browsers, exchange JSON control messages, and relay RTP/RTCP between browsers and server-side application logic.

## Architecture: plugin-based

All actual features live in **server-side plugins** that the browser contacts through Janus. Plugins implement specific applications: echo tests, conference bridges, media recorders, SIP gateways, etc. The Janus core is the plumbing; the plugin is the product.

## Why it matters for Actuate

A frame-transport [[webrtc-deep-dive|WebRTC]] layer for the connector would naturally be a Janus plugin (or a Janus-shaped service) rather than a custom server: the [[webrtc-deep-dive|WebRTC]] handshake, ICE negotiation, and RTP/RTCP relaying are non-trivial and Janus's separation between plumbing (core) and product (plugin) maps cleanly onto an "ingest plugin" Actuate would own.

## Contrast with peer options

- [[2026-05-18_aiortc]] — Python library, not a server. Use for in-process [[webrtc-deep-dive|WebRTC]] where the Python process is the peer.
- [[2026-05-18_pion-webrtc]] — Go library, also not a server. Closer to aiortc but in Go.
- MediaMTX ([[2026-05-18_mediamtx]]) — general media server with [[webrtc-deep-dive|WebRTC]] as one of many supported protocols. Less specialized than Janus but covers more ground.

## Open questions
