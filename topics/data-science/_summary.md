# Data Science Methodology

## Detection Pipeline (Full Stack)

```
Camera Stream (RTSP/SMTP/AILink/Sentinel)
    -> VMS Connector (frame ingestion)
    -> FDMD Motion Detection -> Motion Polygons
    -> YOLO Model Inference (K8s model-svc)
    -> Post-Processing Filters:
        - Stationary Filter (bbox vs motion polygon overlap)
        - Confidence Filter
        - Ignore Zones
        - IOU filter
        - Blacklist filter
    -> Observers (per-product logic):
        - IntruderObserver (frame_thresh consecutive)
        - PersonLoitererObserver (BoTSORT, dwell-time)
        - VehicleLoitererObserver (BoTSORT, dwell-time)
        - LineCrossingObserver (TrajectoryManager, sign-change)
    -> Alert Generation -> S3 + SQS
    -> (Optional) VLM FP Reduction Filter (Qwen3-VL-8B)
```

## Training Infrastructure

### SageMaker Pipeline (Revamp in progress)
- Takes labeled data from **Spektar** (labeling tool)
- Pipeline definition with YOLO architecture + custom augmentation
- Core principle: full traceability (data, parameters, artifacts logged)
- Previous pipeline (YOLOv5 + Neuron compilation) broke with every arch change
- Repo: `ds-sagemaker-yolov5/pipeline-mode`

### Data Management
- **Batch:** Smallest unit (generated by Training Data Sampler)
- **Data Pack:** Logical collection of batches (e.g., construction sites)
- **Data Deck:** Logical collection of data packs
- Provenance: map training runs to compiled data decks

## Motion Detection & Stationary Filter

**Problem:** FDMD was built for continuous RTSP streams, but ~32K cameras run on clip-based connections where frames arrive minutes apart, breaking motion detection assumptions.

**Key fixes needed:**
- Enable cumulative motion blob (MISS-652)
- Skip FDMD for single-frame clips (MISS-630)
- Parameter research and threshold analysis

## Tracking (Loitering/Line Crossing)

**Loitering:** BoTSORT multi-object tracking. Dwell time exceeds configurable threshold. Person uses max_age=5, vehicle max_age=3. Stationary filter on first frame only.

**Line Crossing:** TrajectoryManager (max_link_dist=250px, max_age=15 frames). Sign-change crossing condition with directional filtering. Beta results: **86-98% alert volume reduction** (Edgeworth customer).
