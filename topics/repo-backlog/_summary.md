# Repo Backlog

Cross-repo survey of GitHub issues across major Actuate repos. Complements the Jira-centric workstreams in [[personal-notes/_summary|personal-notes]] by capturing **GitHub-only issues** — including those not assigned to Mark, filed by partners or users, or living in repos he doesn't touch daily.

Populated by [[skill-repo-scan|/repo-scan]], which writes both a dated cross-repo scan note AND per-repo catalog files per run.

## Purpose

The [[automation-jira-sync]] daily job only refreshes *Jira tickets assigned to Mark*. This topic is the GitHub counterpart — surfaces high-impact work and low-hanging fruit that would otherwise go invisible. Items here are *candidates* for pickup, not committed work; promotion to [[mark-todos]] happens through the user's own decision.

## Layout

- `notes/scans/<YYYY-MM-DD>_scan.md` — one note per `/repo-scan` run. Dated snapshot, cross-repo top-5 per bucket. Past days immutable.
- `notes/concepts/<repo>.md` — full per-repo catalog. Auto-refresh block regenerated by `/repo-scan`; hand-maintained `## Curated notes` section preserved.

## Reading a scan note

Each dated scan has two tables:

| Bucket | What's in it |
|--------|-------------|
| 🔥 High Impact | P0/P1/critical/priority-labeled, bug/production/incident/security-labeled, high reactions/comments, recently updated |
| 🧹 Low-Hanging Fruit | `good-first-issue`, `chore`, `docs`, `refactor`, `test`, `cleanup` labels; no assignee; tight scope; clear body |

Direct GitHub URLs in the title column — click through to triage, then (if pursuing) run [[skill-todos-add|/todos-add]] to scaffold a workstream or add to Today's Scope via [[skill-daily-scope|/daily-scope]].

## Stale issues — codebase-scan follow-up (NOT bulk-close)

Issues idle >180d are surfaced in a dedicated section of each per-repo catalog. These are **not** bulk-close candidates. Workflow per stale ticket:

1. `git log --all --grep=<keyword>` + `git log --since=<filedate>` on the relevant subsystem.
2. Comment on the GH issue with commit refs if addressed, OR bump with a "still valid as of <scan date>" note.
3. Close with `addressed by <commit>` only when evidence is explicit.

**Never batch-close without per-ticket evidence.** Many old tickets describe real bugs that predate later refactors.
